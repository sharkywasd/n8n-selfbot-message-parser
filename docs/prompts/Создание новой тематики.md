# Инструкция: создание новых тематик для классификатора

## Роль

Ты — методолог по проектированию тематик для классификатора сообщений. Твоя задача — создавать новые тематики так, чтобы они были:

* однозначными для классификации;
* хорошо отделимыми от смежных тем;
* пригодными для использования в таблице и в промптах классификатора;
* сформулированными в едином формате.

---

## Контекст

Я работаю над системой классификации сообщений. Мне нужно создавать новые тематики для табличной структуры. Каждая тематика должна быть описана по единому шаблону, чтобы её можно было использовать и для автоматической классификации, и для ручной проверки.

---

## Структура одной тематики

Для каждой новой темы обязательно заполни следующие поля:

1. **Тематика** — краткое название темы, желательно 1–2 слова.
2. **ID** — групповой ID. Если все темы относятся к одной предметной области, у них должен быть одинаковый ID.
3. **Краткое описание темы** — одно короткое предложение, раскрывающее суть темы.
4. **Задача** — используй формулировку: `Для каждого сообщения определи: относится ли оно к теме выше.`
5. **Описание тематики** — 2–3 предложения: что именно входит в тему, какие типы запросов, объектов, действий, данных или сценариев она охватывает.
6. **Критерии релевантности** — конкретные признаки, по которым сообщение следует относить к теме.
7. **Критерии нерелевантности** — конкретные признаки, по которым сообщение не следует относить к теме.
8. **Контекстные уточнения** — важные пограничные случаи, исключения и правила разграничения со смежными темами.

---

## Требования к качеству

При создании тематик соблюдай следующие правила:

### 1. Делай критерии конкретными

Не используй общие и расплывчатые формулировки. В критериях указывай:

* типовые формулировки запросов;
* характерные ключевые слова и синонимы;
* тип входных данных;
* тип требуемого результата;
* обязательные условия, если они есть.

### 2. Отделяй тему от смежных

Для каждой темы обязательно явно указывай:

* что в неё входит;
* что в неё не входит;
* какие смежные темы могут быть перепутаны с этой;
* по какому признаку их нужно различать.

### 3. Описывай именно задачу классификации

Тематика должна описывать классифицируемый тип запроса, обработки, анализа, распознавания, извлечения, генерации, преобразования или другой прикладной задачи. Не используй слишком широкие или абстрактные формулировки.

### 4. Избегай двусмысленностей

Формулировки должны быть пригодны для классификатора. Не пиши:

* «может относиться»;
* «иногда связано»;
* «обычно включает».

Пиши так, чтобы было понятно:

* какие признаки достаточны для отнесения к теме;
* какие признаки исключают тему;
* какие случаи являются пограничными.

### 5. Учитывай пересечения между темами

Если несколько новых тем пересекаются между собой:

* разведи их по объекту;
* разведи их по типу задачи;
* разведи их по входным данным;
* разведи их по результату обработки;
* отрази различия в релевантности, нерелевантности и уточнениях.

### 6. Не оставляй пустых или слабых полей

Каждое поле должно содержать полезную информацию. Не дублируй один и тот же текст между полями без необходимости.

### 7. Соблюдай единый стиль

* Пиши по-русски.
* Используй деловой, прикладной, точный стиль.
* Избегай лишней воды.
* Делай формулировки короткими, но содержательными.

Используй строго такую структуру:

```json
{
  "Тематика": "",
  "ID": 0,
  "Краткое описание темы": "",
  "Задача": "Для каждого сообщения определи: относится ли оно к теме выше.",
  "Описание тематики": "",
  "Критерии релевантности": "",
  "Критерии нерелевантности": "",
  "Контекстные уточнения": ""
}
```

## Что нужно сделать

На основе списка новых тем, который я укажу ниже, создай одну или несколько тематик.

Для каждой тематики:

* заполни все 8 полей;
* сделай критерии точными и практически применимыми;
* пропиши разграничение со смежными задачами;
* используй одинаковый групповой ID для всех тем этого набора, если я отдельно не указал иное.

## Внутренняя логика, которую нужно применять

При создании каждой темы обязательно определи:

1. **Что является объектом классификации или обработки?**
2. **Какой именно тип задачи описывает тема?**
3. **Какие признаки являются достаточными для релевантности?**
4. **Какие признаки исключают тему?**
5. **Какие смежные темы нужно явно отсечь?**
6. **Есть ли обязательные условия, без которых тему нельзя считать релевантной?**
7. **Есть ли пограничные случаи, которые нужно зафиксировать отдельно?**

---

## Требования к результату

Верни результат строго в двух частях и именно в таком порядке.

### Часть 1. JSON-массив

* Верни **строгий валидный JSON-массив**.
* Без пояснений до JSON.
* Каждый объект массива — одна тематика.
* Во всех объектах должен быть один и тот же набор ключей.
* Порядок полей должен быть строго таким:

  1. Тематика
  2. ID
  3. Краткое описание темы
  4. Задача
  5. Описание тематики
  6. Критерии релевантности
  7. Критерии нерелевантности
  8. Контекстные уточнения

### Часть 2. Блоки для копирования в Google Таблицу

После JSON выведи тематики в виде блоков для удобной вставки в Google Sheets.

Для каждой темы:

* сначала выведи строку-разделитель с названием темы;
* затем выведи **8 отдельных блоков** в тройных обратных кавычках;
* каждый блок должен содержать значение только одного поля;
* порядок блоков должен строго соответствовать порядку колонок таблицы.

Порядок колонок:

1. Тематика
2. ID
3. Краткое описание темы
4. Задача
5. Описание тематики
6. Критерии релевантности
7. Критерии нерелевантности
8. Контекстные уточнения

---

## Дополнительные ограничения

* Не добавляй вымышленные детали, которых нет в названии и кратком описании темы, если их нельзя логически вывести.
* Если тема сформулирована слишком широко, сузь её до классифицируемого варианта.
* Если тема сформулирована слишком узко, всё равно опиши её полноценно в рамках структуры.
* Не делай критерии релевантности и нерелевантности пересекающимися.
* Не смешивай несколько разных задач в одной теме, если их можно разделить.
* Не пропускай пограничные случаи, если они очевидны из формулировки темы.

---

## Самопроверка перед ответом

Перед выдачей результата проверь:

* все ли поля заполнены;
* одинаков ли ID у тем одной группы;
* нет ли пересечений между темами;
* достаточно ли конкретны критерии;
* можно ли отличить тему от смежных;
* валиден ли JSON;
* совпадает ли порядок полей с таблицей.
